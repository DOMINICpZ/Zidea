# 灵光宝盒 V0.2.0

灵光宝盒是一款基于 Electron + React 开发的桌面端课程管理与学习工具，支持多种媒体格式的课程内容管理、沉浸式播放体验以及课程的导入导出功能。

---

## 一、系统整体功能

### 1.1 核心功能模块

#### 1.1.1 课程管理
- **课程创建**：支持创建多个独立课程，每个课程拥有独立的页面管理
- **课程编辑**：可修改课程名称、描述等元数据信息
- **课程删除**：支持删除课程及其关联的所有页面资源
- **课程列表**：采用卡片式布局展示，支持导入导出功能

#### 1.1.2 页面类型支持
灵光宝盒支持多种课程内容格式：

| 页面类型 | 支持格式 | 说明 |
|---------|---------|------|
| 本地网页 | HTML + 资源文件 | 上传 ZIP 文件自动解压，支持完整网页预览 |
| 外部链接 | HTTP/HTTPS URL | 支持 iframe 内嵌或新窗口打开第三方网站 |
| 图片 | PNG、JPG、JPEG、GIF、WEBP | 支持适应屏幕或原始大小两种显示模式 |
| PDF 文档 | PDF | 内置 PDF 阅读器，支持文档浏览 |
| 视频 | MP4、WebM、OGG | 支持主流视频格式播放，内置播放器控件 |

#### 1.1.3 页面管理
- **拖拽排序**：可视化拖拽调整页面顺序，实时保存排序结果
- **页面预览**：编辑模式下实时预览页面内容效果
- **页面编辑**：支持修改页面名称、类型、外部链接地址等属性
- **批量上传**：ZIP 文件可包含多个页面，自动识别并导入

#### 1.1.4 播放器功能
- **沉浸式播放**：全屏模式播放课程内容，专注学习体验
- **页面导航**：支持上一页/下一页快捷切换，键盘快捷键操作
- **进度记忆**：自动保存学习进度，下次打开自动恢复
- **侧边导航**：可折叠的侧边栏，快速跳转到任意页面

#### 1.1.5 数据导入导出
- **导出课程**：将课程打包为 `.lgbh` 格式文件，包含完整课程数据和资源
- **导入课程**：支持导入 `.lgbh` 或标准 `.zip` 格式的课程文件
- **智能重命名**：导入时自动检测重名课程，提供智能重命名选项

## 二、系统部署方式

### 2.1 环境要求

#### 2.1.1 必需软件
- **Node.js**: 版本 18.0.0 或更高
  - 下载地址: https://nodejs.org/
  - 安装时请确保勾选 "Add to PATH"

#### 2.1.2 支持平台
- Windows 10/11 (x64)
- macOS 10.15+ (Intel/Apple Silicon)
- Linux (Ubuntu 18.04+ 等主流发行版)

---

### 2.2 开发环境部署

#### 步骤 1: 安装项目依赖
```bash
# 在根目录执行，会自动安装主进程和渲染进程的依赖
npm install
```

安装过程说明：
- 根目录依赖：Electron、TypeScript、构建工具等
- 渲染进程依赖：React、Ant Design、状态管理库等
- 预计耗时：3-5 分钟（取决于网络环境）

#### 步骤 2: 启动开发模式
```bash
npm run dev
```

该命令会并行启动：
- **Vite 开发服务器**：提供前端热重载（端口 3000）
- **Electron 主进程**：加载开发环境页面，支持主进程代码热重载

首次启动后，应用窗口会自动打开。开发模式下：
- 前端代码修改后自动刷新
- 主进程代码修改后自动重启 Electron
- 默认开启开发者工具（DevTools）

---

### 2.3 生产环境构建

#### 步骤 1: 执行完整构建
```bash
npm run build
```

构建流程：
1. 编译前端代码（TypeScript + Vite 打包）
2. 编译 Electron 主进程代码
3. 使用 electron-builder 打包为可执行文件

#### 步骤 2: 获取构建产物
构建完成后，安装包输出到 `release/` 目录：

| 平台 | 输出文件 |
|-----|---------|
| Windows | `release/灵光宝盒 Setup {version}.exe` |
| macOS | `release/灵光宝盒-{version}.dmg` |
| Linux | `release/灵光宝盒-{version}.AppImage` |

### 2.5 常见问题排查

#### 问题 1: 提示 "未找到 Node.js"
- 确认 Node.js 版本 >= 18.0.0：`node --version`
- 检查环境变量 PATH 是否包含 Node.js 安装目录
- 重新打开终端窗口后重试

#### 问题 2: 依赖安装失败
- 检查网络连接状态
- 尝试使用国内镜像源：
  ```bash
  npm config set registry https://registry.npmmirror.com
  ```
- 清除 npm 缓存后重试：`npm cache clean --force`

#### 问题 3: 导入课程后无法预览
- 确认使用的是 V0.2.0 及以上版本导出的 `.lgbh` 文件
- 旧版本导出的文件可能存在兼容性问题
- 检查导入后的课程页面列表是否正常显示

#### 问题 4: 构建失败
- 确认已执行 `npm install` 完成依赖安装
- 检查磁盘剩余空间（建议预留 2GB 以上）
- 查看控制台输出的详细错误日志

### 2.6 数据备份与迁移

#### 手动备份
直接复制数据目录即可完整备份：
```bash
# Windows
xcopy "%APPDATA%\灵光宝盒" "D:\备份\灵光宝盒数据\" /E /I

# macOS
cp -r ~/Library/Application\ Support/灵光宝盒 ~/备份/灵光宝盒数据/

# Linux
cp -r ~/.config/灵光宝盒 ~/备份/灵光宝盒数据/
```

#### 数据恢复
将备份的数据复制回原目录即可恢复。

## 三、技术栈说明

### 3.1 前端技术栈
- **框架**: React 18.3.1
- **语言**: TypeScript 5.3
- **UI 组件库**: Ant Design 5.14
- **状态管理**: Zustand 4.5
- **路由**: React Router 6.22
- **构建工具**: Vite 5.1
- **拖拽排序**: @dnd-kit/core + @dnd-kit/sortable

### 3.2 桌面端技术栈
- **框架**: Electron 41.3
- **进程通信**: IPC (contextIsolation + preload)
- **文件压缩**: adm-zip
- **打包工具**: electron-builder 24.9

### 3.3 开发工具
- **类型检查**: TypeScript
- **任务运行**: npm-run-all
- **环境变量**: cross-env
